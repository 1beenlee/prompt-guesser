# Content Ops (Puzzle Library) — 운영 규칙

## 0. 목적
- 퍼즐의 품질/안전/재현성을 유지하면서 운영비 0원으로 큐를 축적한다.

## 1. 퍼즐 생성/큐잉 원칙
- 기본은 자동 생성(추후) + 레포 커밋으로 큐잉
- 전략적 필요 시 운영자가 특정 날짜 퍼즐을 수동 생성/삽입 가능
  - 스폰서십
  - 공휴일/시즌
  - 트렌드/이슈

## 2. 금지/주의 카테고리
- 상표/로고/캐릭터/실존 인물 중심 이미지 금지
- NSFW/폭력/혐오 요소 금지
- 특정 사건사고를 직접 묘사하는 콘텐츠는 보수적으로 운영

## 3. 퍼즐 품질 체크
- 선택지 3개 모두 “겉보기엔 그럴싸”해야 함
- 정답 선택지에는 결정적 디테일 1~2개 포함
- 프롬프트는 1~2줄(5~10초 읽기)

## 4. 에셋 규격
- 이미지: webp(권장), 1024~1536 정사각형
- 용량: 300~700KB 목표(모바일 최적)
- alt 텍스트 포함

## 5. 검증
- PR에서 스키마/경로 검사 통과 필수
